`.github/workflows/` 폴더는 드래그로 안 만들어질 수 있으니, 그 파일 하나는
Add file → Create new file → 파일명 칸에 `.github/workflows/auto-publish.yml` 이라고
직접 입력(슬래시를 치면 폴더가 자동 생성됨) → 내용 붙여넣기 → Commit 하세요.

## API 키 등록 (5분, 이걸 해야 로봇이 글을 씁니다)

1. console.anthropic.com → API Keys → 기존 키(채팅에 노출된 것)는 **Revoke(폐기)**
2. Create Key → 이름 `maison-blog`, 만료 Never → 새 키 복사
3. GitHub `consulting` 저장소 → **Settings** 탭 → 왼쪽 **Secrets and variables** → **Actions**
4. **New repository secret** 클릭
   - Name: `ANTHROPIC_API_KEY` (정확히 이 이름이어야 합니다)
   - Secret: 새 키 붙여넣기
5. Add secret

여기 넣은 키는 화면에 다시 표시되지 않고, 자동화 로봇만 꺼내 씁니다.

## 첫 시동 (테스트)

1. 저장소 → **Actions** 탭 → 왼쪽 "자동 글 발행" 클릭
2. 오른쪽 **Run workflow** 버튼 → Run workflow (수동 실행)
3. 2~5분 뒤 초록 체크가 뜨면 성공. 사이트에 새 글 2편(칼럼 1 + 인사이트 1)이 올라와 있습니다.
4. 실패(빨간 X)하면 로그 화면을 캡처해서 보여주세요.

이후에는 매일 오전 9시대 / 저녁 7시대(한국시간, 매번 조금씩 다름)에 자동 실행됩니다.

## 운영 참고

- **비용**: 글 4편/일 기준 월 8~12만 원 선. console.anthropic.com → Usage에서 확인.
  크레딧이 떨어지면 발행이 조용히 멈추니, 잔액을 가끔 확인하세요. (Auto-reload를 켜면 신경 쓸 일 없음)
- **주제 소진**: 주제 풀 80개는 약 20일 분량입니다. `automation/topics.json`에
  `{"topic": "새 주제", "used": false}` 형식으로 추가하면 됩니다. 저(Claude)에게 "주제 40개 더 만들어줘" 하셔도 됩니다.
- **발행 중단**: Actions 탭 → 자동 글 발행 → 우측 ··· → Disable workflow. 다시 켜려면 Enable.
- **품질 안전장치(내장)**: 지어낸 경험담 패턴·홍보성 표현·중복 제목·분량 미달은 자동 폐기 후 재생성. 3회 실패하면 그 회차는 건너뜁니다(불량 글이 나가느니 안 나가는 게 낫습니다).
- **주말**: 토·일은 일부 회차가 무작위로 쉽니다. 기계적 규칙성을 깨기 위한 의도된 동작입니다.
